**Schritt 4: Crash-Loop-Schutz verifizieren (DMN-02)**

```bash
# Daemon mit kaputter Config crashen lassen, prüfen, dass er NICHT in Endlos-Loop läuft.
# Plan 1 hat in Phase 1 Config-Validation auf process.exit(0) gesetzt → kein Restart bei Config-Error.

# Backup .env, dann Key beschädigen:
cp .env .env.backup
sed -i.tmp 's/^GATHER_API_KEY=.*/GATHER_API_KEY=invalid-test-key-12345/' .env

# Daemon manuell neu starten
launchctl kickstart -k gui/$(id -u)/agency.deepr.gather-apple-music-bridge

# 5–10 Sekunden warten, dann prüfen
sleep 8
launchctl list | grep gather-apple-music
# Erwartet: PID = "-" (nicht laufend), exit code 0 (NICHT 1, weil Config-Validation auf exit(0) sitzt)
# WICHTIG: KeepAlive { SuccessfulExit: false } hält den Daemon down → kein Restart.

tail -5 ~/Library/Logs/gather-bridge.log
# Erwartet: pino "fatal" Log mit dem Config-Validation-Error, dann nichts mehr.

# Restore .env
mv .env.backup .env
rm -f .env.tmp
launchctl kickstart -k gui/$(id -u)/agency.deepr.gather-apple-music-bridge
sleep 3
launchctl list | grep gather-apple-music
# Erwartet: PID wieder Zahl (Daemon läuft mit valider Config wieder).
```
